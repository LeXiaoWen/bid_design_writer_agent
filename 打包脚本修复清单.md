# Windows 打包脚本修复清单

修复日期：2026-07-07

---

## 1. `package.json` — 跨平台脚本模式有缺陷

**问题**：`dev:agent` 和 `install:agent` 使用了 `./scripts/run-agent.sh || scripts\\run-agent.bat` 的 hack 写法。
在 Windows 上每次都会先尝试执行 `.sh` 文件，cmd.exe 无法识别，输出错误信息后才 fallback 到 `.bat`。

```
'./scripts/run-agent.sh' 不是内部或外部命令，也不是可运行的程序
```

**修复**：创建了 `scripts/run-agent.js` 和 `scripts/setup-agent.js` 两个跨平台 JS 包装器，
检测 `process.platform` 后自动执行对应平台的原生脚本。

**涉及文件**：
- `package.json:12` — dev:agent 脚本
- `package.json:28` — install:agent 脚本
- `scripts/run-agent.js` — 新增
- `scripts/setup-agent.js` — 新增

---

## 2. `scripts/run-agent.bat` — 环境变量未加引号

**问题**：`%AGENT_HOST%` 和 `%AGENT_PORT%` 在 uvicorn 命令中裸用，若变量含空格会导致命令解析错误。

**修复**：变量引用加双引号。

**涉及文件**：
- `scripts/run-agent.bat:7`

---

## 3. `scripts/build-agent.mjs` — shell:true 导致带空格路径崩溃

**问题**：`run()` 函数在 Windows 上设置 `shell: true`，Node.js 将命令和参数拼接为字符串传给 `cmd.exe`。
若 `.agent-venv` 路径或项目根目录包含空格（如 `C:\Program Files`），`cmd.exe` 解析出错。

**修复**：改为 `shell: false` + `windowsHide: true`。所有命令均为 Python 可执行文件（`.exe` 直接路径
或 `py`/`python` 命令名），无需 shell 介入。

**涉及文件**：
- `scripts/build-agent.mjs:14-22`

---

## 4. `scripts/run-electron-builder.mjs` — Node.js 路径空格阻塞

**问题**：原脚本通过 `electron-builder.cmd` 调用 electron-builder，使用 `shell: false`。
在 Node.js 装在 `C:\Program Files\nodejs\`（含空格）的 Windows 系统上，electron-builder 内部
spawn 子进程时路径未被正确引用，始终报 `'C:\Program' 不是内部或外部命令`。

**修复**：绕过 `.cmd` 包装器，直接用 `process.execPath`（node.exe 全路径）运行 electron-builder
的 `cli.js`，并创建 `.local-bin/node.bat` 代理脚本前置到 PATH 作为 fallback。

**涉及文件**：
- `scripts/run-electron-builder.mjs` — 重写
- `.local-bin/node.bat` — 新增（Node.js 路径代理）

---

## 5. `packaging/icons/icon.ico` — 假的 ICO 文件

**问题**：`icon.ico` 实际是 **1254×1254 的 PNG 图片**（1.7MB），并非真正的 ICO 格式。
electron-builder 校验时报错：`Icon is not a valid ICO file`。

**修复**：用 Pillow 将 PNG 转为标准 ICO 格式（256×256，85KB），并保留 `packaging/icons/icon.png` 备用。

**涉及文件**：
- `packaging/icons/icon.ico` — 替换为真正的 ICO
- `packaging/icons/icon.png` — 新增（原始 PNG 备份）
