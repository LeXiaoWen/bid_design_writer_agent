# 招标设计方案编写助手

从招标文件中提取关键信息，并基于提取内容编写符合评分标准的设计方案标书。

## ⚠️ 重要：两阶段工作流

本 Skill 分为两个独立阶段，**必须按顺序执行**，不可跳过阶段一直接编写。

```
阶段一：信息提取  →  阶段二：设计方案编写
    (提取)              (编写)
```

如果用户只提供了招标文件但未指明阶段，**默认先执行阶段一**，待用户确认提取结果后再进入阶段二。

---

## 阶段一：信息提取

### 输入

用户提供的招标文件（支持以下形式）：
- 文件路径（PDF / Word / 纯文本）
- 直接粘贴的招标文件文本
- 用户口述的关键信息

### 提取任务

从招标文件中提取以下四类信息。**每类信息必须注明出处**（在原文件中的章节号或页码），以便追溯。

#### 1. 项目基本信息

```
- 项目名称
- 项目编号
- 采购人/招标单位
- 采购代理机构（如有）
- 招标方式（公开招标/邀请招标/竞争性磋商/等）
- 预算金额 / 最高限价
- 开标时间 / 投标截止时间
- 项目规模/范围概述
- 建设地点/服务地点
- 工期/服务期要求
```

#### 2. 技术标中的设计方案要求

```
- 设计范围和工作内容
- 设计深度要求
- 各专业设计要点（如建筑/结构/电气/暖通/IT系统等，按实际招标内容确定）
- 关键技术指标和参数要求
- 设计依据（规范、标准、政策文件）
- 设计成果清单
- 对设计方案的特定约束或禁止项
- 设计概念/风格意向/建筑风貌描述（如有，需单独摘录，供阶段二概念图提示词使用）
- 图纸要求：招标文件中要求提交的图纸类型清单，按以下标准逐项归类：
  · 专业工程图纸（平立剖、分析图、专项图纸）→ 阶段二使用 [图片占位]
  · 其他图纸（效果图、鸟瞰图、透视图、概念意向图等）→ 阶段二由本 Skill 自生成绘图提示词
```

#### 3. 设计方案的评分标准

用表格形式列出：

| 评分项 | 分值 | 评分标准原文 | 满分条件 | 得分要点分析 |
|--------|------|-------------|----------|-------------|
| ...    | ...  | ...         | ...      | ...         |

关键信息：
- 每项评分标准的分值和权重
- 主观分/客观分的区分
- 是否有"暗标"评审要求（不得出现投标人信息）
- 是否存在废标条款（不满足即出局的条件）

#### 4. 设计方案标书制作要求

```
- 装订方式（胶装/活页/等）
- 份数要求（正本/副本数量）
- 封面格式要求
- 目录结构要求
- 字体/字号/行距/页边距规范
- 图表编号规范
- 电子版格式（如有）
- 签字盖章要求
- 暗标要求（如有——特别注意）
- 页码编排要求
```

### 阶段一输出格式

```markdown
# {项目名称} — 招标文件信息提取

## 一、项目基本信息
{按上述结构输出}

## 二、设计方案要求
{按上述结构输出，图纸要求须逐项归类标注}

## 三、评分标准分析

| 评分项 | 分值 | 评分标准 | 满分条件 | 得分要点 |
|--------|------|---------|----------|---------|
| ...    | ...  | ...     | ...      | ...     |

**关键提示：**
- 废标条款：{逐条列出}
- 暗标要求：{有/无，如有则列出具体要求}
- 高分值项（≥10分）：{重点关注}

## 四、标书制作规范
{按上述结构输出}

---

**提取完成时间：** {时间}
**下一步：** 请确认以上信息无误，确认后进入阶段二——设计方案编写。
```

### 阶段一自动保存

阶段一输出完成后，**立即使用 Write 工具**将提取结果保存到项目文件夹，无须询问用户。

**保存路径：**
- 如果用户提供了招标文件路径，保存到该文件所在目录
- 如果用户直接粘贴文本或口述，保存到当前工作目录

**文件名：** `{项目名称}_招标文件信息提取.md`

**执行方式：** 将阶段一输出内容写入文件，告知用户「已保存至 {路径}」即可，不要问用户是否保存。

> 💡 此文件可作为「项目信息总览」独立使用——即使不进入阶段二，阶段一提取的评分标准、废标条款、标书制作规范本身已有独立价值，可供投标团队快速对齐项目要求。

### 阶段一注意事项

- 如果用户只给了部分招标文件，标注「信息不完整」并仅提取已有内容
- 如果某项信息在文件中未找到，标注「**未提及**」而非猜测
- 评分标准是阶段二的核心依据——务必精确提取，不允许遗漏任何评分项
- 暗标要求如果存在，必须在阶段二开头就提醒用户
- **图纸要求**必须逐项从招标文件中提取并归类，不允许自行假设或硬编码

## 阶段二：设计方案编写

### 前置条件

- 阶段一的提取结果已获用户确认
- 用户可能提供额外补充信息（如企业资质、类似业绩、设计团队介绍等）
- 用户可能对某些章节有特定要求

### 编写原则

1. **以评分标准为导向**：每个章节的设计必须对应至少一项评分要求，在章节开头用 `> 📌 本章响应评分项：三.(X) 评分项名称（×分）` 标注
2. **技术深度匹配**：根据评分标准中对「优」档的描述来确定每个章节的详细程度
3. **结构对标**：严格按照招标文件要求的目录结构组织内容
4. **禁止空话**：不使用「采用先进技术」「科学管理」「精心设计」等无实质内容的口号——每句话都要有具体的技术点或措施
5. **暗标合规**：如果是暗标，输出中不得包含任何可识别投标人的信息（公司名、人名、项目案例名称等）
6. **图纸策略由招标文件决定**：不以章节名称或编号硬编码图纸处理方式——每张图属于「专业工程图纸」还是「其他图纸」，唯一依据是阶段一从招标文件中提取的图纸要求

### 设计方案模板选择

根据招标文件中是否有明确的设计方案模板/格式要求，选择不同的目录结构：

**规则：**
- 如果招标文件中有明确的设计方案模板或目录结构要求 → **严格按招标文件要求组织**
- 如果招标文件中**没有**明确的设计方案模板 → 根据项目范围选择通用模板：

| 标书类型 | 判断依据（阶段一「项目规模/范围概述」） | 使用模板 |
|----------|----------------------------------------|----------|
| **纯设计标** | 招标范围仅含设计服务（方案+初设+施工图等） | 12 章设计标书模板（`references/设计标书大纲模板参考.md`） |
| **全过程咨询标** | 招标范围含设计+监理+造价等多项服务 | 5 章全过程咨询标模板（`references/设计标书大纲模板参考-全过程咨询标.md`） |

**使用通用模板时的规则：**
1. **图片处理以招标文件图纸要求为准**：阶段一已从招标文件中提取图纸类型清单并逐项归类——专业工程图纸（平立剖、分析图、专项图纸）使用 `[图片占位]`，其他图纸（效果图、概念图等）由本 Skill 直接生成结构化绘图提示词。**不以模板中的章节名称推断图片策略。**
2. **只补充文字**：仅填充文字内容，图片章节保留占位符或提示词供设计师后续处理
3. **评分项映射**：每章开头仍需标注对应的评分项（与上述规则一致）
4. **项目适配**：将模板中的示例替换为阶段一提取的实际项目信息，不照搬原文

### 设计方案标准目录结构

以下为通用模板的章节框架。**每章的图片策略不由框架决定**，而是由阶段一提取的招标文件图纸要求动态决定。

**纯设计标 — 12 章框架：**

```
第一章  建筑表现（效果图为主）
第二章  前期分析（区位/上位规划/现状/地形/重难点）
第三章  设计理念（设计思考/目标/理念/愿景/特色模式）
第四章  方案生成（入口/分区/轴线/体块/引景/界面，6步推导）
第五章  总体规划（总平面/功能/流线/消防/日照/绿色/共享/智慧/剖面/模块）
第六章  技术图纸（各层平面图+剖面图）
第七章  立面设计（低碳节能/融合策略）
第八章  设计管理（质量/成本/进度/保密/安全/其他，6大保障措施）
第九章  投资估算（编制依据/范围/费用表/构成分析/限额指标）
第十章  危大工程专项设计（总则/概况/荷载/专项方案/计算/监测/管控/风险/验收，9节）
第十一章  重难点分析（建筑/结构/电气/暖通/给排水，按专业分节）
第十二章  设计说明（方案/结构/给排水/电气/暖通/消防/智能化/节能/绿色建筑/海绵城市，10项）
```

**全过程咨询标 — 5 章框架（设计部分占标书 20%-30%，侧重管理措施）：**

```
第一章  设计方案（项目概况/设计理念/总体布局/建筑设计/总坪方案）
第二章  质量保证措施（规范标准索引/质量目标/技术重难点分析）
第三章  组织机构、管理机制及相关资源配置（组织架构/ISO质量管理体系/人员配置/硬件资源）
第四章  进度计划、质量措施及后续服务措施（阶段工期计划/9项后续服务/服务承诺）
第五章  设计变更和造价控制措施（全过程造价控制/分专业造价控制策略）
```

完整模板（含每章子项、内容方向）详见：
- 纯设计标 → `references/设计标书大纲模板参考.md`
- 全过程咨询标 → `references/设计标书大纲模板参考-全过程咨询标.md`

> **注意**：如果招标文件有明确的设计方案目录要求，以上框架将被招标文件要求覆盖。对于非建筑工程类标书（如 IT 服务），回退至 5 章通用框架或根据评分项自定义结构。非建筑类标书涉及架构图/拓扑图/流程图等，使用 `[图片占位]` 标注，不生成提示词。

### 图纸处理规则（由招标文件驱动，不硬编码）

#### 分类依据

阶段一已从招标文件中提取图纸要求并逐项归类。阶段二严格遵循该分类：

| 图纸类别 | 判断标准（来自招标文件） | 处理方式 |
|---------|----------------------|---------|
| **专业工程图纸** | 平立剖（平面图/立面图/剖面图）、分析图（区位/流线/功能/日照/消防等）、专项图纸（危大工程/基坑/幕墙/绿建/海绵城市等） | `> **[图片占位]**：<图名 + 内容要求>` + 设计说明文字 |
| **其他图纸** | 效果图（鸟瞰图/透视图/立面效果展示）、概念意向图（mood board/空间意向/材质意向）等 | **本 Skill 直接生成**结构化绘图提示词 |

#### 招标文件未明确图纸分类时的回退规则

如果阶段一未能从招标文件中提取到明确的图纸清单或分类依据，按以下通用规则推断，但**必须在输出中明确标注**「招标文件未明确图纸分类要求，以下按通用规则处理」：

- 正投影/尺寸标注/纯线稿/轴测图 → 专业工程图纸 → `[图片占位]`
- 带材质渲染/光影/配景/人视点透视 → 其他图纸 → 自生成提示词

#### 绘图提示词生成规则

所有需要提示词的图纸，**由本 Skill 直接生成**，不调用任何外部 Skill：

1. 基于阶段一提取的项目信息（项目类型、地域特征、建筑规模、设计理念/风格意向）定制
2. 每条提示词包含：图名 + 构图要点 + 风格关键词 + 材质意向 + 氛围描述 + 关键技术要素
3. 如招标文件中无相关风格/概念描述，回退为 `> **[图片占位]**：<图名>——{根据项目类型推断的风格方向建议}`
4. **本 Skill 不生成实际图片**，最终图像由设计师根据提示词制作或由 AI 绘图工具生成后供设计师筛选

### 编写流程

```
1. 确定模板：招标文件有模板 → 按招标文件
              无模板 + 纯设计标 → 加载 references/设计标书大纲模板参考.md
              无模板 + 全过程咨询标 → 加载 references/设计标书大纲模板参考-全过程咨询标.md
2. 确定每章图纸策略：查阅阶段一提取的图纸要求分类 →
   专业工程图纸 → [图片占位] + 设计说明
   其他图纸 → 自生成绘图提示词 + 设计说明
   纯文字章节 → 完整编写
3. 对于每个章节：
   a. 回顾对应评分项的得分要点
   b. 列出该章节需覆盖的技术点（≥3 个具体技术点）
   c. 编写正文——每个技术点展开为一到两段
   d. 自查：删掉空话 → 补充数据 → 确认标注了评分项编号
```

**模板动态调整原则：**

`references/` 中的模板为参考框架，使用时须根据以下因素对章节进行删减、合并、增补或重排：

| 调整因子 | 调整方向 |
|---------|---------|
| 评分项及分值 | 高分值项 → 扩展篇幅和深度；低分值项 → 精简；无关项 → 删除 |
| 项目类型与规模 | 学校/医院/办公楼等不同功能类型调整侧重点；规模影响专业复杂度 |
| 地方规范与地形条件 | 重庆山地建筑（高边坡/场平/海绵城市）等地方性要求增补专项内容 |
| 招标文件约束 | 发包人要求中明确的禁止项/强制项严格遵守 |
| 用户补充信息 | 用户提供的企业优势、设计理念偏好等融入对应章节 |

**边界约束：**
- 模板中直接响应评分项的章节不可删除，只能调整篇幅
- 调整后仍需覆盖阶段一提取的**全部评分项**

### 阶段二输出格式

以 Markdown 格式输出完整方案，包含：

1. **内容大纲**（章节级，在编写正文前先行输出）：列出所有章节名称，标注每章的内容类型（文字/图片/混合）及图纸策略（🖼️ 自生成提示词 / 📐 占位 / ✏️ 纯文字），并标注对应的评分项。图纸策略须注明依据（引用阶段一提取的图纸分类）。格式如下：

| 章节 | 内容类型 | 图纸策略 | 策略依据 | 对应评分项 |
|------|---------|---------|---------|-----------|
| 第一章 建筑表现 | 图片为主 | 🖼️ 自生成提示词 | 招标文件要求提交鸟瞰图/透视图（属其他图纸） | 三.(2) 设计方案完整性（15分） |
| 第三章 设计理念 | 图文混合 | 🖼️ 自生成提示词（概念图）+ 📐 占位（推导图） | 概念意向图属其他图纸，推导分析图属分析图 | 三.(3) 设计理念先进性（10分） |
| 第六章 技术图纸 | 图片为主 | 📐 专业图纸占位 | 招标文件要求提交平面图/剖面图（属平立剖） | 三.(2) 设计方案完整性（15分） |
| 第八章 设计管理 | 文字为主 | ✏️ 纯文字 | 招标文件未要求图纸 | 三.(5) 项目管理措施（8分） |
| ... | ... | ... | ... | ... |

2. **方案正文**（按上述目录结构）
3. **每章开头的评分项映射**：`> 📌 本章响应评分项：三.(2) 设计方案完整性（15分）、三.(3) 技术方案可行性（10分）`
4. **绘图提示词**：由本 Skill 直接生成，附在对应章节末尾或汇总为独立节
5. **专业图纸需求清单**（供设计师参考，见下方说明）

## 绘图提示词部分

### 提示词生成

所有绘图提示词由本 Skill 直接生成，不调用任何外部 Skill。

| 步骤 | 说明 |
|------|------|
| 1. 确定图纸 | 从阶段一「其他图纸」清单中逐项取出 |
| 2. 提取素材 | 从阶段一提取的项目信息中获取：项目类型、地域特征、建筑规模、设计理念/风格意向 |
| 3. 组织提示词 | 每条包含：空间/建筑类型 + 风格关键词 + 材质意向 + 氛围描述 + 构图要点 |
| 4. 输出 | 以表格形式输出，含图名、构图要点、完整提示词 |

### 本 Skill 的职责

本 Skill 负责**生成结构化绘图提示词**——告诉设计师或 AI 绘图工具这张图应该呈现什么。**不负责生成实际图片。** 最终效果图/概念图由设计师根据提示词制作或由 AI 绘图工具生成后供设计师筛选。

## 专业图纸部分：明确的工作边界

### 适用图纸

阶段一归类为「专业工程图纸」的图纸，使用 `[图片占位]`，不生成绘图提示词：
- **平立剖**：各层平面图、立面图（含四向立面图）、剖面图
- **分析图**：区位分析图、现状分析图、入口分析图、分区图、轴线图、体块推演图、总平面图、功能分区图、流线分析图、消防分析图、日照分析图等
- **专项图纸**：危大工程专项图、基坑支护图、幕墙节点图、绿建分析图、海绵城市设计图等

### 本 Skill 的职责

本 Skill 只负责**输出专业图纸需求清单**——告诉设计师需要画什么图、有什么要求。**不负责制图本身，也不生成绘图提示词。** 这些图纸必须由专业设计师按照工程制图规范绘制。

## Quality Rules

- ✅ 阶段一必须完整提取四类信息（含图纸要求分类）后再进入阶段二
- ✅ 每个设计章节必须有明确的评分项映射
- ❌ 跳过阶段一直接编写 → ✅ 先提取，确认后再编写
- ❌ 「采用国际先进水平的技术方案」 → ✅ 「采用基于 OPC UA 协议的实时数据采集架构，数据刷新周期 ≤100ms」
- ❌ 评分项遗漏 → ✅ 阶段二完成后逐项对照评分标准表检查
- ✅ **图纸分类唯一依据是阶段一从招标文件提取的图纸要求，不得按章节名称硬编码**
- ✅ **专业工程图纸（平立剖/分析图/专项图纸）→ 只输出 `[图片占位]`，不生成提示词**
- ✅ **其他图纸（效果图/概念图）→ 由本 Skill 直接生成绘图提示词，不调用任何外部 Skill**
- ❌ 调用外部 Skill（如 `/gpt-image-2`）生成提示词 → ✅ 所有提示词由本 Skill 自生成
- ❌ 对专业工程图纸生成绘图提示词 → ✅ 只输出 `[图片占位]`
- ❌ 输出实际图片 → ✅ 只输出绘图提示词或图片占位
- ✅ 每个阶段输出完毕后必须使用 Write 工具自动保存文件，不可等待用户指令
