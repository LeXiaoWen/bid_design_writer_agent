# 招标设计方案编写助手

从招标文件中提取关键信息，并在有参考材料时建立“招标约束 + 已有方案事实 + 评分响应”的一致性底稿，再编写符合评分标准的设计方案标书。

## 核心定位与交付边界

本 Skill 只处理**建筑设计技术标书**。AI 负责形成可直接审核的完整文字初稿；设计师负责核验项目事实、专业结论、图纸和投标人履约承诺，而非重新拼凑或润色正文。

每个项目须按阶段交付以下四项成果：

1. **项目信息总览**：项目基本信息、设计范围、硬约束、废标条款、暗标要求和制作规范。
2. **评分标准拆解**：评分表、满分条件、得分要点和评分交付矩阵。
3. **对应内容大纲**：按评分项列出章节，明确每章是文字、提示词还是专业图纸占位。
4. **完整标书初稿**：完整文字方案、绘图提示词、专业图纸需求清单和来源一致性附表；默认交付 Markdown，招标文件或用户要求 Word 时同步交付 `.docx`。

**图片边界**：本 Skill 不调用出图能力，也不生成实际图片。效果图和概念意向图只生成提示词；流线、功能、日照、消防、总平等分析图以及平立剖、专项图只输出制图需求和占位，由设计师完成。

## ⚠️ 重要：两阶段工作流

本 Skill 分为两个独立阶段，**必须按顺序执行**，不可跳过阶段一直接编写。

```
阶段一：信息提取  →  阶段二：设计方案编写
    (提取)              (编写)
```

如果用户只提供了招标文件但未指明阶段，**默认先执行阶段一**。

🔴 **CHECKPOINT**：阶段一完成后**必须暂停**，等待用户确认提取结果无误，方可进入阶段二。禁止在用户未确认的情况下自动进入阶段二。

## 阶段一：信息提取

### 输入

用户提供的招标文件（支持以下形式）：
- 文件路径（PDF / Word / 纯文本）
- 直接粘贴的招标文件文本
- 用户口述的关键信息

用户还可提供或在招标文件同目录中保留以下**同项目参考材料**：
- 已有方案/概念方案/方案深化文本（如 `方案.pdf`）
- 历史投标文件、施工图设计大纲、技术标样本
- 规划条件通知书、红线 CAD、地形图、地勘、甲方标准、项目会议纪要

### 参考材料发现、使用与权威性规则（必做）

1. **发现**：若输入为文件路径，先列出其所在目录中名称含“方案”“投标”“施工图设计大纲”“任务书”“规划条件”“红线”“地勘”“会议纪要”的 PDF/Word/Excel/CAD 文件；不得扫描上级目录或无关项目目录。
2. **登记**：阶段一必须输出“参考材料台账”，列明文件名、用途、可提取信息和是否实际采用。用户明确排除的文件不得读取。
3. **权威性排序**：
   - A 级：招标文件、澄清/补遗、正式规划条件、采购人书面确认；用于硬约束、评分、工期、范围和最终指标。
   - B 级：用户明确为“当前/已确认”的同项目方案、甲方任务书、会议纪要；用于项目事实、图文结构、产品/立面/地下室策略和图纸清单。
   - C 级：未确认版本的同项目方案、历史投标文件、施工图设计大纲、企业模板；只用于组织方式、颗粒度、质量/进度/成本/服务措施，不得直接复制投标人名称、人员、资质、业绩、承诺时限或未核实的技术指标。
4. **冲突阻断**：A 级与 B/C 级对面积、容积率、产品类型/面积、车位、工期、设计范围或图纸要求冲突时，必须建立“数据冲突清单”。阶段二可继续使用 A 级值；不得使用冲突的 B/C 级值，也不得把其写入提示词、指标表或图纸需求清单。若 A 级未给出该字段，则写“待采购人/甲方确认”。
5. **引用边界**：参考材料用于提高项目化程度，不改变招标文件的强制要求、评分项和暗标规则。若参考材料为历史投标文件，暗标项目中不得复用其任何可识别信息。

### 实体污染与规范有效性审计（B/C 级材料必做）

真实投标文件也可能包含跨项目复制残留，不能因其已成册就视为事实来源。读取 B/C 级材料后，必须执行以下审计：

1. **实体核对**：提取项目名称、建设单位、地点、地块号、建筑类型、服务对象、既有校园/园区名称、周边要素、专业范围，与 A 级项目事实表逐项比对。
2. **跨项目词扫描**：扫描不同城市/区县、不同业主、不同学校/园区、不同项目类型（如中小学/医院/住宅/高层）和不属于本项目的规范主管部门。命中后列入“污染词清单”。
3. **数值与承诺核对**：所有面积、层数、高度、容积率、成本单价、预警阈值、响应时限、人员数量、服务频次必须有 A/B 级出处；C 级数值和承诺仅可作为“待投标人确认”的候选项。
4. **规范核对**：标准名称、编号、年份、适用地区和适用建筑类型须与 A 级要求或现行有效规范一致；引用不完整、过期、跨地区或不适用规范时，不得复用。
5. **处置**：污染内容禁止进入正文、提示词、图纸清单和模块卡；仅当其抽象为不含实体和数值的流程/版式方法时才可保留。

### 真实投标反向拆解（存在 B/C 级参考材料时必做）

不要只记录“可用内容”。必须从真实方案/技术标中提取以下四类可复用资产，并标注其来源页码：

1. **评分交付矩阵**：每个评分项在真实材料中使用了什么文本、图纸/效果图、表格/节点、页组和证据。区分“评分要求的必要交付”与“仅用于表现的装饰页”。
2. **视觉资产矩阵**：图名、图纸类型、表达目的、构图/版式、关键标注、所服务评分项、可复用条件。若真实材料是 PDF/幻灯片式技术标，还须记录画幅、横竖版、每页信息密度、章节扉页样式和“单页主证据”结构。对每张效果图还须拆出“视点/朝向—前景—中景主体—背景/城市界面—体量与材料锚点—光照/季节—页内后加文字或图签”，并区分“渲染图本身”与“排版叠加信息”。只沉淀图文组合和表达方法，不复制项目专有图像、品牌或未确认指标。
3. **措施拆解表**：施工图大纲中的质量、进度、成本、服务措施，拆为“控制对象—触发节点—责任角色—输出记录—验收/闭环方式”五要素；缺任一要素的文字不得作为可复用措施模板。
4. **可复用模块卡**：将项目专有数据剥离后，按适用条件写成模块卡；具体格式见 `references/可复用模块卡片.md`。不得把某一项目的章节顺序、主题名称、面积、户型、材料或承诺时限设为全项目默认值。

阶段一仅做提取、反向拆解和冲突标注，不据此跳过确认点。

### 提取任务

从招标文件提取以下四类信息，并结合已登记参考材料补充“可复用项目事实”。**每类信息必须注明出处**（文件名 + 章节号或页码），以便追溯。

#### 1. 项目基本信息

```
- 项目名称
- 项目编号
- 采购人/招标单位
- 采购代理机构（如有）
- 招标方式（公开招标/邀请招标/竞争性磋商/等）
- 预算金额 / 最高限价
- 开标时间 / 投标截止时间
- 项目规模/范围概述
- 建设地点/服务地点
- 工期/服务期要求
```

#### 2. 技术标中的设计方案要求

```
- 设计范围和工作内容
- 设计深度要求
- 各专业设计要点（如建筑、结构、给排水、电气、暖通、景观、室内等，按实际招标内容确定）
- 关键技术指标和参数要求
- 设计依据（规范、标准、政策文件）
- 设计成果清单
- 对设计方案的特定约束或禁止项
- 设计概念/风格意向/建筑风貌描述（如有，需单独摘录，供阶段二概念图提示词使用）
- 图纸要求：招标文件中要求提交的图纸类型清单，按以下标准逐项归类：
  · 专业工程图纸（平立剖、分析图、专项图纸）→ 阶段二使用 [图片占位]
  · 其他图纸（效果图、鸟瞰图、透视图、概念意向图等）→ 阶段二由本 Skill 自生成绘图提示词
```

#### 3. 设计方案的评分标准

用表格形式列出：

| 评分项 | 分值 | 评分标准原文 | 满分条件 | 得分要点分析 |
|--------|------|-------------|----------|-------------|
| ...    | ...  | ...         | ...      | ...         |

关键信息：
- 每项评分标准的分值和权重
- 主观分/客观分的区分
- 是否有"暗标"评审要求（不得出现投标人信息）
- 是否存在废标条款（不满足即出局的条件）

#### 4. 设计方案标书制作要求

```
- 装订方式（胶装/活页/等）
- 份数要求（正本/副本数量）
- 封面格式要求
- 目录结构要求
- 字体/字号/行距/页边距规范
- 图表编号规范
- 电子版格式（如有）
- 签字盖章要求
- 暗标要求（如有——特别注意）
- 页码编排要求
```

### 阶段一自动保存

阶段一输出完成后，立即使用当前环境可用的文件写入能力将提取结果保存到项目文件夹，无须询问用户。

**保存路径：**
- 如果用户提供了招标文件路径，保存到该文件所在目录
- 如果用户直接粘贴文本或口述，保存到当前工作目录

**文件名：** `{项目名称}_招标文件信息提取.md`

**执行方式：** 将阶段一输出内容写入文件，告知用户「已保存至 {路径}」即可，不要问用户是否保存。

> 💡 此文件可作为「项目信息总览」独立使用——即使不进入阶段二，阶段一提取的评分标准、废标条款、标书制作规范本身已有独立价值，可供投标团队快速对齐项目要求。

🛑 **STOP**：保存完成后，输出「阶段一提取完毕。请确认以上信息是否准确完整，确认后我将进入阶段二编写设计方案。」**等待用户明确确认**，不可自动继续。

### 阶段一异常处理

| 触发条件 | 一线修复 | 仍失败兜底 |
|---------|---------|-----------|
| PDF 无法解析/渲染 | 尝试 `python3 -c "import PyPDF2; ..."` 提取文本；若纯文本提取失败，尝试 `strings` 命令提取可读内容 | 告知用户「PDF 无法自动解析」，请用户手动粘贴招标文件关键章节（评分标准、技术要求的页码） |
| 用户提供的文件路径不存在 | 提示用户检查路径拼写，给出当前工作目录下可见的 pdf/doc/docx 文件列表供选择 | 请用户拖拽文件到对话框 或 直接粘贴文件内容 |
| 招标文件为扫描件/图片型 PDF（无可提取文本） | 告知用户「此 PDF 为扫描件，无法自动提取文字」，列出需要用户提供的关键信息清单（评分标准、图纸要求、标书规范） | 用户逐项口述或粘贴关键信息，标注来源为用户口述 |
| 招标文件内容超过 50 页 | 分段提取：先提取评分标准和废标条款（阶段二核心依据），再提取项目信息和设计要求的摘要，最后提取标书规范 | 告知用户「文件较长，以下为优先级提取结果」，标注未完整覆盖的章节 |

### 阶段一注意事项

- 如果用户只给了部分招标文件，标注「信息不完整」并仅提取已有内容
- 如果某项信息在文件中未找到，标注「**未提及**」而非猜测
- 评分标准是阶段二的核心依据——务必精确提取，不允许遗漏任何评分项
- 暗标要求如果存在，必须在阶段二开头就提醒用户
- **图纸要求**必须逐项从招标文件中提取并归类，不允许自行假设或硬编码
- **同目录有参考材料时，不得仅按通用模板编写**：必须完成参考材料台账、项目事实摘录和数据冲突清单后才可进入阶段二。
- 未获确认的 B/C 级数值可作为“待确认依据”列出，但不得覆盖 A 级值，不得混写为已确认项目指标。

## 阶段二：设计方案编写

### 前置条件

- 阶段一的提取结果已获用户确认
- 用户可能提供额外补充信息（如企业资质、类似业绩、设计团队介绍等）
- 用户可能对某些章节有特定要求
- 如存在同项目参考材料：已完成参考材料台账与数据冲突清单；未解决的冲突已被标记为“待确认”，不会写入确定性内容。

### 阶段二异常处理

| 触发条件 | 一线修复 | 仍失败兜底 |
|---------|---------|-----------|
| 模板文件（`references/` 下的 `.md`）不存在或不可读 | 列出缺失的模板文件名，告知用户；按评分项自定义章节结构（不依赖模板） | 以评分标准为唯一依据自建目录框架，每项评分对应一个章节，标注「模板缺失，已按评分标准自定义结构」 |
| 文件保存失败（权限/磁盘空间/路径不存在） | 检查目标目录是否存在，不存在则尝试创建；权限不足则尝试保存到当前工作目录 | 告知用户「自动保存失败：{错误原因}」，将完整内容直接输出到对话中，请用户手动保存 |

### 编写原则

1. **以评分标准为导向**：每个章节的设计必须对应至少一项评分要求，在章节开头用 `> 📌 本章响应评分项：三.(X) 评分项名称（×分）` 标注
2. **技术深度匹配**：根据评分标准中对「优」档的描述来确定每个章节的详细程度
3. **结构对标**：严格按照招标文件要求的目录结构组织内容
4. **禁止空话**：不使用「采用先进技术」「科学管理」「精心设计」等无实质内容的口号——每句话都要有具体的技术点或措施
5. **暗标合规**：如果是暗标，输出中不得包含任何可识别投标人的信息（公司名、人名、项目案例名称等）
6. **图纸策略由招标文件决定**：不以章节名称或编号硬编码图纸处理方式——每张图属于「专业工程图纸」还是「其他图纸」，唯一依据是阶段一从招标文件中提取的图纸要求
7. **项目化优先**：有 B 级同项目方案时，优先复用其已确认的章节叙事、场地特征、图纸名称、产品/地下室/立面策略和视觉方向；不得用通用“高端生态”“现代立面”等词替代已知项目事实。
8. **事实可追溯**：所有面积、产品户型、车位、密度、高差、材料、节点、进度和服务时限必须在 A/B 级来源中有出处；无法确认时写“待确认”，不补造数值。
9. **历史材料去标识化**：C 级材料只能提炼方法、流程和检查项；涉及企业组织、资质、人员、案例、项目承诺或专利时，必须删除或转为“由投标人按实际能力填写”。

### 评分驱动双册结构（优先用于“方案深化设计大纲 + 施工图设计大纲”）

#### A 册：方案深化设计文本

以评分中的方案深化项为章节骨架，而非按施工图管理章节堆叠。以下六章是“住宅、已有概念方案、方案深化评分包含场地/总平/立面/地下室/产品/鸟瞰”时可调用的**模块组合**，不是全项目默认目录；仅当适用条件和评分项匹配时采用：

```
第一章  项目背景与项目理解（区位、现状、场地价值、设计主题）
第二章  规划设计（方案生成、彩色总平、结构、流线、消防、剖面）
第三章  立面与材料深化（立面策略、材料、节点、落地性与成本）
第四章  地下室优化（范围、车位/柱网/坡道、机房/管综、归家流线、指标）
第五章  产品优化（户型配比、典型户型、空间亮点、设备与施工图条件）
第六章  总体鸟瞰与效果回顾（只放招标要求的效果图/提示词与图纸说明）
```

必须逐项映射：项目理解、场地分析图、彩色总平、已有方案优化（外立面 + 地下室）、户型优化、总体鸟瞰。若招标文件要求不同，以评分项替换章节，而不是保留无关章节。

#### B 册：施工图设计大纲

以评分中的施工图大纲五项为一级目录：

```
第一章  设计工作重点、难点分析及合理化建议
第二章  设计质量保障措施（设计输入、条件图、校审、专业对图、审图闭环）
第三章  设计进度保障措施（倒排计划、里程碑、风险与纠偏）
第四章  成本控制保证体系与措施（建筑、结构、机电、区域容量、地下室/立面）
第五章  服务承诺及后续保障措施（报审、交底、变更、现场、验收；承诺须由投标人确认）
```

若 C 级施工图大纲提供质量体系、专业对图清单、会签节点、进度分解、成本台账和服务流程，可抽取其**方法和检查项**填入对应章节；不得复制企业名称、人员履历、企业制度名称或未经投标人确认的时限承诺。

**施工图措施写法（强制）**：每一项质量、进度、成本或服务措施均须写清五要素：

| 要素 | 必须回答的问题 |
|------|----------------|
| 控制对象 | 控制什么图纸、指标、接口或现场问题？ |
| 触发节点 | 在什么阶段、什么条件或什么风险出现时启动？ |
| 责任角色 | 谁组织、谁复核、谁确认？仅写角色，不虚构人员姓名。 |
| 输出记录 | 形成什么表、清单、纪要、图纸或台账？ |
| 验收/闭环 | 由谁按何种标准确认关闭；未关闭如何升级？ |

例如，“专业对图”必须写为：在施工图完成约 75% 且各专业提资冻结后，由项目负责人组织建筑、结构、机电和总图开展叠图核对，形成《专业接口问题清单》；问题按责任专业、关闭日期、复核人销项，复核通过后在图纸首页会签。具体比例、角色配置和会签形式须以投标人体系及项目要求确认后采用。

### 图纸处理规则（由招标文件驱动，不硬编码）

#### 分类依据

阶段一已从招标文件中提取图纸要求并逐项归类。阶段二严格遵循该分类：

| 图纸类别 | 判断标准（来自招标文件） | 处理方式 |
|---------|----------------------|---------|
| **专业工程图纸** | 平立剖（平面图/立面图/剖面图）、分析图（区位/流线/功能/日照/消防等）、专项图纸（危大工程/基坑/幕墙/绿建/海绵城市等） | `> **[图片占位]**：<图名 + 内容要求>` + 设计说明文字 |
| **其他图纸** | 效果图（鸟瞰图/透视图/立面效果展示）、概念意向图（mood board/空间意向/材质意向）等 | **本 Skill 直接生成**结构化绘图提示词 |

#### 招标文件未明确图纸分类时的回退规则

如果阶段一未能从招标文件中提取到明确的图纸清单或分类依据，按以下通用规则推断，但**必须在输出中明确标注**「招标文件未明确图纸分类要求，以下按通用规则处理」：

- 正投影/尺寸标注/纯线稿/轴测图 → 专业工程图纸 → `[图片占位]`
- 带材质渲染/光影/配景/人视点透视 → 其他图纸 → 自生成提示词

#### 绘图提示词生成规则

所有需要提示词的图纸，**由本 Skill 直接生成**，不调用任何外部 Skill：

1. 先建立**视觉锚点表**，再写提示词。基于阶段一提取的 A/B 级项目事实（项目类型、地域特征、已确认总平/体量/材料、设计理念/风格意向）定制；有已确认方案或同项目效果图时，提示词必须与其空间锚点和视觉语言一致。C 级效果图只可借鉴构图、镜头和信息层级，不得借用其建筑数量、层数、面积、地名、道路距离或材料定案。
2. 每条效果图提示词必须按以下六段组织，缺一不可：
   - **任务与视点**：图名、观察者位置/朝向、相机高度或鸟瞰类型、主视对象；
   - **空间构图**：前景—中景主体—背景/城市界面，以及道路、入口、水体、山体、绿地等已确认关系；
   - **建筑锚点**：仅写已确认的体量关系、层级、屋顶、立面材料、开窗/遮阳、连廊等；未确认内容写“待建筑专业确认”，不得虚构；
   - **场景与光线**：时段、天气、季节、灯光层级、植被和人物活动；
   - **表达方式**：写实建筑可视化、构图比例、留白/信息焦点；渲染图本身不含文字；
   - **禁止项**：文字、校名/项目名、标志、图签、页码、水印、夸张异形体、与总平矛盾的道路/建筑/山水，以及未经确认的功能和技术设施。
3. 真实投标效果图中常见的标题、项目名、页码、定位小图、结论文字属于**后期版式叠加**，不得要求绘图工具生成；在“视觉版式页组脚本”中另行列为设计师排版任务。
4. 未经 A/B 级依据核实的楼层、面积、建筑数量、道路距离、精确相机角度、材料品牌、景观设施和设备不得写入提示词；如确需表现，标记为“待建筑专业确认”的可选项。不得以“8K”“获奖级”等空泛参数替代空间构图要求。
5. 如招标文件及 A/B 级材料均无相关风格/概念描述，回退为 `> **[图片占位]**：<图名>——待甲方确认视觉方向`，不得按项目类型臆造风格。
6. **本 Skill 不生成实际图片**，最终图像由设计师根据提示词制作或由 AI 绘图工具生成后供设计师筛选。

#### 效果图硬性数量与视点校验

当招标文件对效果图数量、夜景、鸟瞰、单体、主入口、主轴、节点或城市道路视域有明确要求时，必须建立“效果图合规清单”：

| 招标要求 | 最低数量 | 计划图名/视点 | 提示词编号 | 与平面/总图一致性校核 |
|----------|----------|---------------|------------|------------------------|
| ... | ... | ... | ... | 待建筑专业确认 |

- 每项硬性效果图要求均须有一条独立提示词或明确图片占位；不得以一张图同时虚报多个必须视点。
- 效果图的建筑数量、层数、入口位置、道路视角、功能分区和立面材料必须与 A 级总图/平面或已确认 B 级方案一致；未确认时标为“待确认”，不虚构。
- 每条提示词均须引用视觉锚点表，且在交付前由设计师对照总平、平立面或已确认方案核查“视点、前中后景、体量关系、材料和城市界面”；仅风格接近不视为通过。
- 真实投标中有效的“单页主证据 + 一句结论 + 一张主图”可作为版式方法复用，但不得照搬其项目名称、主题、形象和视觉素材。

**模板动态调整原则：**

`references/` 中的模板为参考框架，使用时须根据以下因素对章节进行删减、合并、增补或重排：

| 调整因子 | 调整方向 |
|---------|---------|
| 评分项及分值 | 高分值项 → 扩展篇幅和深度；低分值项 → 精简；无关项 → 删除 |
| 项目类型与规模 | 学校/医院/办公楼等不同功能类型调整侧重点；规模影响专业复杂度 |
| 地方规范与地形条件 | 重庆山地建筑（高边坡/场平/海绵城市）等地方性要求增补专项内容 |
| 招标文件约束 | 发包人要求中明确的禁止项/强制项严格遵守 |
| 用户补充信息 | 用户提供的企业优势、设计理念偏好等融入对应章节 |

**边界约束：**
- 模板中直接响应评分项的章节不可删除，只能调整篇幅
- 调整后仍需覆盖阶段一提取的**全部评分项**

## Quality Rules

- ✅ 阶段一必须完整提取四类信息（含图纸要求分类）后再进入阶段二
- ✅ 每个设计章节必须有明确的评分项映射；未被招标范围、提交要求或评分项触发的可选章节不得输出

- ✅ 评分分别要求方案深化与施工图大纲时，优先按双册结构输出；通用12章不得挤占或替代评分直接要求的章节
- ✅ A 级招标/澄清/正式规划资料优先于 B/C 级方案和历史材料；冲突指标不得混用
