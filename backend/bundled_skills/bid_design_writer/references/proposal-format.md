# 设计方案标书编写规范

> 配套 SKILL.md 阶段二使用。保证多项目输出格式一致。

## 通用写作规范

### 语言风格
- 使用客观、专业的技术语言
- 禁用主观空洞词汇：先进、一流、卓越、完美、精心、全力
- 每句话必须含可验证的具体信息（技术名称/数据/标准编号/流程步骤）

### 段落结构
- 每段开头一句话概括本段主旨
- 段中展开具体技术内容（≥2 个具体技术点）
- 段尾呼应评分标准中的得分要点

### 禁用词清单

| 禁用词 | 替换为 |
|--------|--------|
| 采用先进技术 | 采用 {具体技术名称}，实现 {具体指标} |
| 科学管理 | 按 {标准编号} 建立管理体系，含 {N} 个控制节点 |
| 精心设计 | 针对 {具体场景}，在 {具体环节} 采取 {具体措施} |
| 严格把关 | 设置 {N} 级评审，每级检查 {具体内容} |
| 全方位服务 | 提供 {具体服务项1}、{具体服务项2}、{具体服务项3} |
| 积极响应 | 接到通知后 {N} 小时内响应，{N} 小时内到场 |

## 章节编写指引

### 第一章：项目概述与设计理念
- **目标**：证明你深度理解项目需求
- **必含**：对招标文件关键需求的回应性叙述（≥3 处引用招标文件原文）
- **避坑**：不要复述招标文件——要展示你对需求背后意图的洞察

### 第二章：专业技术方案
- **目标**：展示技术实力——这是高分值章节
- **必含**：每个专业至少 3 个具体技术措施，每个措施附参数/标准/流程
- **结构建议**：按招标文件所列专业逐个展开，不遗漏任何一个

### 第三章：设计质量保障
- **目标**：证明设计过程可控
- **必含**：引用至少 2 个国家标准或行业规范编号
- **亮点**：增加设计评审的具体流程（节点/参与角色/检查项）

### 第四章：进度计划与保障措施
- **目标**：证明能按时交付
- **必含**：里程碑列表（含日期）、人员投入（人数×天数）、风险应对预案（≥3 个风险点）
- **可用表格或描述性甘特图**

### 第五章：服务承诺与增值服务
- **目标**：加分项——拉开与竞争对手的距离
- **必含**：具体的响应时间承诺（数字）、具体的增值服务项
- **避坑**：不承诺超出企业能力范围的服务

## 评分项映射标注格式

每个章节开头使用以下标注格式：

```markdown
> 📌 本章响应评分项：
> - 三.(2) 设计方案完整性 — 15分
> - 三.(3) 技术方案可行性 — 10分
```

## 暗标注意事项

如果招标文件要求暗标评审（盲评），必须遵守：

1. 全文不出现投标人名称、简称、英文缩写
2. 不出现投标人过往项目名称（可用「某省级政务云项目」替代）
3. 不出现投标人获奖、专利、资质等可追溯信息
4. 不出现投标人人员姓名（可用「项目负责人」「结构专业负责人」替代）
5. 格式严格按暗标要求（统一字体字号，封面无标识等）
